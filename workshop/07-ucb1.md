## The Key Insight

UCB1 solves the explore-exploit dilemma with one elegant principle: **be optimistic about uncertain options**.

If you're unsure about an arm, assume it's good. The only way to reduce uncertainty is to try it. Once you've tried it enough, the optimism bonus shrinks and the true performance dominates.

## The Formula

```
UCB(arm) = mean_reward(arm) + c × √(ln(total_pulls) / arm_pulls)
                 ↑                        ↑
           exploitation              exploration
```

- **First term (exploit)**: Pick arms with high average reward
- **Second term (explore)**: Pick arms with few pulls (high uncertainty)
- **c**: Controls the tradeoff. Higher c = more exploration

### How It Evolves Over Time

```
Round 1-5:   Exploration bonus dominates (all arms have few pulls)
Round 5-20:  Bonus shrinks for frequently-played arms, stays high for rare ones
Round 20+:   Best arm dominates, poor arms rarely played
Round 100+:  Almost pure exploitation, occasional exploration of neglected arms
```

### UCB Values Over Time

```
UCB Score
  5 │  ●                        ← Arm 2 (untried): high uncertainty
    │
  4 │       ●                   ← Arm 3 (tried once): still uncertain
    │
  3 │            ●──────●       ← Arm 0 (best so far): selected!
    │                 mean=2.8
  2 │                 ●──●      ← Arm 1 (tried many times): low UCB
    │                  mean=2.1
  1 │
    └────────────────────────── Rounds
      1    5    10   15   20

As an arm is pulled more, its confidence interval shrinks.
The exploration bonus (second term) decreases with more pulls.
```

Unlike Explore-Then-Exploit, UCB1 **never fully stops exploring**. The exploration term shrinks but never reaches zero. If conditions change and a previously-bad arm improves, UCB1 will eventually notice.

## Why √(ln(n) / n_i)?

The exploration bonus `√(ln(n) / n_i)` comes from **Hoeffding's inequality** — a concentration bound from probability theory. It says: with probability at least `1 - 1/n^(2c²)`, the true mean of arm `i` is within this bound of the observed mean.

Informally: "the true reward is probably no higher than mean + bonus." By playing the arm with the highest upper bound, you're being **optimistic** — acting as if each arm is as good as it could plausibly be.

## Code Walkthrough

```python
def select_arm(self) -> int:
    # Play each arm once first
    for i in range(self.n_arms):
        if self.counts[i] == 0:
            return i

    # Compute UCB for each arm
    ucb_values = []
    for i in range(self.n_arms):
        mean = self.rewards[i] / self.counts[i]
        exploration = self.c * math.sqrt(math.log(self.total_pulls) / self.counts[i])
        ucb = mean + exploration
        ucb_values.append(ucb)

    return argmax(ucb_values)
```

**Initialization**: Play each arm exactly once. Without at least one observation, we can't compute a mean.

**Selection**: Compute UCB for every arm, pick the highest. Deterministic — given the same history, always picks the same arm.

**Update**: Simple running average:
```python
self.counts[arm] += 1
self.rewards[arm] += reward
# mean = self.rewards[arm] / self.counts[arm]
```

### Reward Computation

The example uses a composite metric to compute reward:

```python
def compute_composite(signals):
    """Weighted combination of all available metrics."""
    weights = {"SCAN_FREQUENCY": 0.4, "RETENTION_RATE": 0.3, "FEEDBACK_QUALITY": 0.2}
    score = 0.0
    for key, weight in weights.items():
        metric = signals.metrics.get(key)
        if metric and metric.confidence != "low":
            score += weight * metric.value
    return score
```

The reward for each arm pull is the delta in this composite score, normalized to [0, 1]:
```python
reward = max(0, min(1, 0.5 + (current_composite - previous_composite)))
```

This normalization is important — UCB1 assumes rewards in [0, 1].

## UCB1 vs Thompson Sampling

| Property | UCB1 | Thompson Sampling |
|----------|------|-------------------|
| Selection | Deterministic | Stochastic |
| Foundation | Frequentist (confidence bounds) | Bayesian (posterior sampling) |
| Theoretical regret | Proven optimal bounds | Proven optimal bounds |
| Empirical performance | Good | Often better |
| Tuning | c parameter | Prior (usually Beta(1,1)) |
| When arms are similar | Explores more (bonus stays high) | Converges faster |

**Recommendation**: Try both. UCB1 is easier to debug (deterministic). Thompson Sampling often finds the best arm faster.

## Tuning the Exploration Parameter c

| c Value | Behavior |
|---------|----------|
| 0.5 | Low exploration — converges fast but may miss the best arm |
| 1.0 | Balanced — good default for Arena |
| √2 ≈ 1.41 | Theoretically optimal for [0,1] rewards |
| 2.0+ | High exploration — tries many arms, slow to converge |

For Arena, where experiments run for weeks and each round takes minutes, `c=1.0` is a good starting point.
